How to contribute

Wir freuen uns über jede Beteiligung an dem Projekt. Sei es für kleinere Bug-Fixes oder komplette neue Features

Das Projekt basiert auf den Sprachen Typescript (Angular Framework) und Java (SpringBoot). Hierbei kann im Frontend oder auch im Backend untersützt werden.

Aber auch außerhalb der eigentliche Webseite kann untersützt werden. Beispielsweise freuen wir uns auch über Mithilfe bei der Dokumentation, welche stets erweitert und verbessert werden muss.

Reporting An Issue

Bugs oder Verbesserungen können über GitHub Issues gestellt werden. Hierbei können Frontend Issues hier und Backend Issues hier gestellt werden. Falls die Zugehörigkeit des Problems nicht eindeutig sein sollte, stelle die Issue im Frontend. Gegebenenfalls werden wir mitteilen, ob es sich um ein Frontend-Problem oder Backend-Problem handelt.
Bitte achtet beim Erstellen darauf, dass das Problem so genau wie möglich beschrieben ist. Falls es zu einer Fehlermeldung kommt, fügt diese auch hinzu. Dadurch wird es für die Bearbeiter einfacher, einen Fehler zu finden.

Contributing to the backend

Falls ihr Änderungen im Backendcode vornehmen möchtet, haltet euch bitte an folgendes:
  • Wenn die Änderungen größer sind und nicht nur ein kleiner Bugfix, erstelle bitte passende UnitTests zum Testen des hinzugefügten Codes.
  • Wenn es sich um eine komplett neues Feauture handelt, haltet euch bitte an den hexagonalen Architekturstil
  • Achte darauf, dass keine unnötigen Änderungen hinzukommen.
  • Halte dich an folgende Prinzipien: Naming-Convention, SOLID-Prinzip, DRY-Prinzip, Error-Handling
  • Prüfe, ob die Dokumentation bezüglich deiner Änderungen angepasst werden muss. Falls ja, stelle hierzu in dem Doku-Repo einen passenden Merge-Request
  • Merge-Requests, welche noch in Bearbeitung sind, mit DRAFT markieren.

Contributing to the frontend

Falls ihr Änderungen im Frontendcode vornehmen möchtet, haltet euch bitte an folgendes:
  • Achte darauf, dass keine unnötigen Änderungen hinzukommen.
  • Halte dich an folgende Prinzipien: Naming-Convention, SOLID-Prinzip, DRY-Prinzip, Error-Handling
  • Prüfe, ob die Dokumentation bezüglich deiner Änderungen angepasst werden muss. Falls ja, stelle hierzu in dem Doku-Repo einen passenden Merge-Request
  • Merge-Requests, welche noch in Bearbeitung sind, mit DRAFT markieren.
  • Prüfe vor dem Stellen des Merge-Requests, dass durch die Änderungen keine Fehlermeldung in der Console auftritt.
  • Nutze falls möglich die Features des Angularframeworks um den Code nicht unnötig aufzublähen
